Higher-order programming and type inference//高阶编程与类型接口

推荐阅读:

  • Haskell趣学指南, chapter 6

Anonymous functions//匿名函数

假如我们想写一个函数:

greaterThan100 :: [Integer] -> [Integer]

它仅保留一个Integer数组中的大于100的元素，比如:

greaterThan100 [1,9,349,6,907,98,105] = [349,907,105]

以目前的知识，我们可能会写出这样的函数:

gt100 :: Integer -> Bool
gt100 x = x > 100

greaterThan100 :: [Integer] -> [Integer]
greaterThan100 xs = filter gt100 xs

但有一个地方让人很不舒服，为了一个简单的x > 100，我们还要把它写成一个完整的函数、给它取一个名字，它的定义用的字比逻辑本身还要多——而花这么多功夫定义的函数可能只会被用一次... 太麻烦了。在这里，我们可以使用匿名函数(anonymous function)，或者叫lambda函数来替代他:

greaterThan100_2 :: [Integer] -> [Integer]
greaterThan100_2 xs = filter (\x -> x > 100) xs

\x -> x > 100就是一个匿名函数，它没有名字，只负责接受一个变量x然后返回x是否大于100的结果，\符号标记了一个匿名函数（它看起来就像是缺了条腿的\(\lambda\)）

匿名函数也可以有多个参数，比如:

Prelude> (\x y z -> [x,2*y,3*z]) 5 6 3
[5,12,9]

不过，在上面greaterThan100的例子中，我们有比用匿名函数更好的写法:

greaterThan100_3 :: [Integer] -> [Integer]
greaterThan100_3 xs = filter (>100) xs

(>100)是一个操作符节(operator section): 如果?是一个操作符，那么(?y)就是一个\x -> x ? y的简写，(y?)就是一个x -> y ? x的简写。换句话说，操作符节允许我们部分调用一个操作符到两个参数中的一个上，然后返回这个已经被部分调用的、只能再接受一个参数的函数，作为我们此处的匿名函数。这里还有一些例子:

Prelude> (>100) 102
True
Prelude> (100>) 102
False
Prelude> map (*6) [1..5]
[6,12,18,24,30]

译注: 记住，操作符也是函数，operator section只不过是对这些函数部分调用的例子而已

Function composition//函数组合

在继续读下去之前，你能写出一个类型如下的函数吗？

(b -> c) -> (a -> b) -> (a -> c)

我们试一试，它需要接受两个参数，两个参数都是函数；它的返回值是另一个函数

foo f g = ...

在...的位置，我们需要写一个类型是a -> c的函数，我们当然可以用匿名函数来写:

foo f g = \x -> ...

显然，x具有类型a，现在在....的位置，我们需要写一个类型为c的表达式，我们恰好有一个叫g的、类型是a -> b的函数和一个叫f的、类型是b -> c的函数，所以这样写肯定没错:

foo :: (b -> c) -> (a -> b) -> (a -> c)
foo f g = \x -> f (g x)

Quick quiz: 为什么(g x)要加括号

不过，我们上面写了那么多有什么用呢。foo是一个有任何实际意义的函数吗？还是说只是一个强行举的让我们熟悉类型的例子？

实际上，这里的foo就是一个函数组合器，foo f g代表将f和g两个函数组合在一起，就像是数学里的\((f\circ g)(x)=(f(g(x)))\)，在Haskell中，已经自带了一个函数组合运算符(.)，可以通过f . g实现foo f g的效果: 先用输入调用g，再用这个结果调用f

函数组合可以帮我们写出清晰而优雅的代码，它也很符合我们wholemeal编程的品味，特别是在我们需要对一个数据结构应用一连串连续的变换函数的时候，比如:

myTest :: [Integer] -> Bool
myTest xs = even (length (greaterThan100 xs))

这就是一串连续调用，为了保证调用顺序正确，我们必须用很多组括号来表示连续调用的函数，利用函数组合，我们可以将它改写成:

myTest' :: [Integer] -> Bool
myTest' = even . length . greaterThan100

这个版本就易读多了: myTest'就像一个流水线，将三个小函数组装在一起。这个例子也展示了为什么函数组合看上去是倒着的(写在.后面的函数先执行)，因为函数应用过程就是自右向左的

我们来仔细看看.函数的类型是什么，问问ghci，它告诉我们:

Prelude> :t (.)
(.) :: (b -> c) -> (a -> b) -> a -> c

等等，(a -> c)的括号去哪了？

Currying and partial application//柯里化和部分调用

回忆一下那些我们觉得很奇怪的接受多个参数的函数的类型:

f :: Int -> Int -> Int
f x y = 2*x + y

为什么不是f :: Int Int -> Int呢，为什么多了一个箭头？

我曾经保证过:

现在，是时候看背后的原因了: 所有Haskell的函数都只接受一个参数。什么鬼？f难道不是一个接受两个参数的函数吗？实际上并不是，它只接受一个Int作为参数，然后返回一个类型为Int -> Int的函数；这个函数又接受一个Int作为参数，然后返回最终结果。实际上，我们可以等价地把这个函数定义写成:

f' :: Int -> (Int -> Int)
f' x y = 2*x + y

我们发现函数定义里的箭头是右结合(associate to the right)的，W -> X -> Y -> Z等价于W -> (X -> (Y -> Z))，因此用括号将函数定义里最右侧的箭头两侧包裹起来总是无所谓的

另一方面，函数应用则是左结合(left-associative)的，f 3 2实际上指的是(f 3) 2，这也符合我们刚才说的，f接受单个参数并返回一个函数: 我们先对3调用f，得到了一个类型为Int -> Int的中间函数，它的实际效果是接受一个Int并返回它加6的结果。然后我们对2调用这个中间函数，得到了一个Int类型的、值为8的返回值。正因为左结合使得左侧的括号可以去除，我们才能把(f 3) 2这个调用写成f 3 2，给我们提供一个“多参数函数”的使用方式

同理，“多参数”的匿名函数\x y z -> ...其实就是\x -> (\y -> (\z -> ...))的语法糖

我们定义的f x y z = ...也是f = \x -> (\y -> (\z -> ...))的语法糖

我们也可以改写之前实现的函数组合器，把\x -> ...从等号的右边移动到左边:

comp :: (b -> c) -> (a -> b) -> a -> c
comp f g x = f (g x)

这种将多参数函数表达成单参数函数，并且返回一个函数的做法叫柯里化(currying)，它以英国(Wiki说是美国)数学家和逻辑学家哈斯克尔·柯里(Haskell Curry, 1900 - 1982)的名字命名，没错，Haskell也是以他命名的。将多参数函数表示为单参数、返回函数的函数的方法实际上是Moses Schönfinkel首次提出的，所以我们可能应该称其为schönfinkeling，只不过因为柯里对此也有贡献，人们已经习惯用currying称呼这个方法了

如果我们真的想表达一个包含两个参数的函数，我们可以用二元组作为其参数:

f'' :: (Int, Int) -> Int
f'' (x,y) = 2*x + y

你可以认为它接受了两个参数(虽然实际上还是一个)，为了在(Int, Int) -> Int和Int -> Int -> Int这两种形式间转换，基础库提供了两个叫curry和uncurry的函数:

curry :: ((a,b) -> c) -> a -> b -> c
curry f x y = f (x,y)

uncurry :: (a -> b -> c) -> (a,b) -> c
uncurry f (x,y) = f x y

uncurry在你有一个二元组并且希望对其调用函数的时候很实用，比如:

Prelude> uncurry (+) (2,3)
5

Partial application

Haskell的函数都是柯里化的，这使得部分调用(partial application)非常容易，部分调用不过是用一部分参数调用一个多参数函数，然后得到一个接受剩下参数的函数——正因为Haskell中没有多参数函数，所以任何函数都是可以部分调用的

不过Haskell的语法导致对除了第一个参数以外的参数部分调用比较困难，只有中缀操作符是例外，我们刚才已经提到了像(>100)或(100>)这样部分调用运算符的例子，它的左右两个参数都可以用来部分调用。不过实际应用中，这个限制并不很明显，好好设计函数的参数顺序使得函数更容易被部分调用也是一种艺术: 参数需要按可变性从小到大排序(the arguments should be ordered from from “least to greatest variation”)，也就是说最开始的参数往往都是一样的，而越往后的参数越容易不同

Wholemeal programming

让我们用一个例子回顾之前学的东西，顺便展示一下wholemeal的编程风格，看下面定义的foobar函数:

foobar :: [Integer] -> Integer
foobar []     = 0
foobar (x:xs)
  | x > 3     = (7*x + 2) + foobar xs
  | otherwise = foobar xs

这个定义非常直白，但是不够Haskell，问题出在:

  • 一次性做了太多工作
  • 在太低的层次(low-level)上工作，比如遍历数组这种事情也自己实现了一遍

不需要考虑我们需要对每个元素做什么操作，我们可以思考对整个输入做一串连续的变换，然后得到以下函数:

foobar' :: [Integer] -> Integer
foobar' = sum . map (\x -> 7*x + 2) . filter (>3)

在这种定义下，foobar'就是一个三个函数构成的流水线: 先丢掉所有不大于3的元素，然后对剩下的元素应用一些数学运算，最后求和。在这个例子里，map和filter都是部分调用的，比如filter的类型是:

(a -> Bool) -> [a] -> [a]

用(>3)调用它会得到一个类型为[Integer] -> [Integer]的函数，这个函数刚好可以与另一个接受[Integer]的函数做组合

这种定义一个完全不引用其参数的函数的编程方法，与其说是告诉程序该做什么，不如说是在描述问题是什么，这种风格也叫Pointfree Style。就像我们看到的，这种风格非常优美，甚至有些人会要求你时刻保证使用Pointfree的编程风格，不过过犹不及。#haskell的IRC频道里的lambdabot包含一个@pl指令，可以将普通函数转换成Pointfree形式，比如:

@pl \f g x y -> f (x ++ g x) (g y)
join . ((flip . ((.) .)) .) . (. ap (++)) . (.)

显然，这个例子里的pointfree可不是一种改进

Folds

我们还有一个针对列表的递归模板需要考虑: folds，下面是一系列类似的针对列表的模板: 它们都通过某种方式将整个列表的元素“压缩”成一个最终值

sum' :: [Integer] -> Integer
sum' []     = 0
sum' (x:xs) = x + sum' xs

product' :: [Integer] -> Integer
product' [] = 1
product' (x:xs) = x * product' xs

length' :: [a] -> Int
length' []     = 0
length' (_:xs) = 1 + length' xs

这些函数的共性是什么，区别又是什么？我们依然可以抽取它们的共性，来帮助我们编写高阶函数

fold :: b -> (a -> b -> b) -> [a] -> b
fold z f []     = z
fold z f (x:xs) = f x (fold z f xs)

可以看fold是如何将[]用f,z和(:)替换的:

fold f z [a,b,c] == a `f` (b `f` (c `f` z))

从这个视角来看，你可能就能找到将fold推广到除列表以外数据结构的方法了

现在我们可以用fold来重写sum', product', length'了:

sum''     = fold 0 (+)
product'' = fold 1 (*)
length''  = fold 0 (\_ s -> 1 + s)

其中，(\_ s -> 1 + s)可以被替换成(\_ -> (1+))甚至是(const (1+))

当然，fold也已经被标准Prelude实现了，函数名是foldr，foldr的参数顺序与我们实现的有小小的不同，不过本质完全一样，以下是一些利用foldr实现的函数:

length :: [a] -> Int
sum :: Num a => [a] -> a
product :: Num a => [a] -> a
and :: [Bool] -> Bool
or :: [Bool] -> Bool
any :: (a -> Bool) -> [a] -> Bool
all :: (a -> Bool) -> [a] -> Bool

同时，Prelude也提供了foldl，l意味着从左开始，就比如:

foldr f z [a,b,c] == a `f` (b `f` (c `f` z))
foldl f z [a,b,c] == ((z `f` a) `f` b) `f` c

不过，如果你真的想用foldl的功能，那请用Data.List提供的foldl'，因为两者效果相同而后者更高效
