Rust Quiz #1

Answer: The program is guaranteed to output: 112

  • 在宏里，$(...)*代表接受0或多个被包裹的Designators(相应的，$(...)+则是一个或多个)，也可以在其中插入分隔符的pattern，以下是一些例子

    ($($s:stmt)*)           matches ({} {} {})   // {} is stmt
($x:expr, $($y:expr)+)  matches (1, 1 1)     // the last two `1` matches $y twice
($x:expr, $($y:expr),+) matches (1, 1, 1)    // almost same as previous one, but has `,` in pattern

    因此，题中的宏能匹配多个statement，它们之间不需要分隔符

  • $(...)<Token>*语法用于展开上条中通配符收集的Designators，以题目中的宏为例，展开的例子如下(来自quiz的官方解答)

    Suppose we call this macro with three of the statements shown above as input.

    m! {
    struct S { x: u64 }
    let mut s = S { x: 1 }
    s.x + 1
}

    The macro expands to:

    { stringify!(struct S { x: u64 }); 1 }
    << { stringify!(let mut s = S { x: 1 }); 1 }
    << { stringify!(s.x + 1); 1 }

    Each of the stringifys expands to a string literal:

    { "struct S { x: u64 }"; 1 }
    << { "let mut s = S { x: 1 }"; 1 }
    << { "s.x + 1"; 1 }

  • 由于展开后的每个block stmt都是一个返回1的expr，因此上面的展开结果等价于1 << 1 << 1，即2^(3-1)=4，即该宏会返回2^(n-1)，n是调用宏时匹配到的语句个数，因此需要考虑题目中三处调用的语句数目

  • 考虑题目中给的三处宏调用

    • return || true 是单条语句，语义为返回一个函数closure: || true
    • (return) || true 是单条语句，语义为(return) logic-or true，(return)是一个表达式
    • {return} || true 是两条语句{return}和|| true，因为前者是单独的块语句

  • 因此答案为112

Rust Quiz #3

Answer: The program is guaranteed to output: 32

const声明的常量的每一次使用都会用其内联值替代，因此原程序等价为

fn main() {
    let v = &mut (S { x: 2 });
    v.x += 1;
    (S { x: 2 }).x += 1;
    print!("{}{}", v.x, (S { x: 2 }).x);
}

Rust Quiz #4

Answer: The program is guaranteed to output: 54

  • 在rhs部分，..是一个操作符RangeFull

  • 在lhs部分，..是模式匹配的占位符

  • 因此最终执行的print语句为

    print!("{}", b"066"[..][1]);  // outputs ascii value of '6'
